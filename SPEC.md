# VoiceToFile 产品规格文档

> 小宇宙播客订阅转文字工具 | 本地 Flask Web UI

---

## 1. 项目概述

**项目名称**：VoiceToFile
**仓库**：`https://github.com/benjem-wu/voicetofile`
**分支策略**：`main` = 稳定版

### 目标

用户在小宇宙上关注了 100~200 个播客，每日人工检查一次：将新episode下载、转写为 TXT、删除音频文件，只保留文字稿。

### 核心约束

- **纯人工触发**，无定时自动任务
- 每集转写完成后**立即删除音频文件**
- 不处理付费内容（只展示，跳过下载）
- 支持单集手动添加（不依赖订阅流程）

---

## 2. 功能架构

### 2.1 两种工作模式

#### 模式 A：播客订阅（批量获取）

```
输入播客链接/ PID
    ↓
抓取播客最近 15 集（xyzcdn.net/podcast/{pid}）
    ↓
展示列表（名称 / 发布日 / 时长 / 付费标识）
    ↓
用户勾选要下载的集数（付费集不可勾选）
    ↓
逐集下载 → 转写 → 保存 TXT → 删除音频
```

#### 模式 B：手动添加单集

```
输入单集链接（https://www.xiaoyuzhoufm.com/episode/{eid}）
    ↓
解析音频 URL，直接下载
    ↓
转写 → 保存 TXT → 删除音频
```

### 2.2 重新处理

已完成的集（`done_deleted` 状态）可重新触发：
→ 重新下载 + 重新转写 → 生成新 TXT（覆盖）

## 3. 数据获取

### 3.1 API / 页面

| 用途 | URL | 说明 |
|------|-----|------|
| 播客 15 集列表 | `https://www.xyzcdn.net/podcast/{pid}` | JSON-LD，含 name/datePublished/timeRequired/description |
| 播客主页（取名称） | `https://www.xiaoyuzhoufm.com/podcast/{pid}` | HTML `<title>` 或 JSON-LD |
| 单集详情 | `https://www.xiaoyuzhoufm.com/episode/{eid}` | HTML JSON-LD，含音频 URL |
| 单集 JSON 数据 | `https://www.xiaoyuzhoufm.com/_next/data/-GOav0dS9wDlfSnB05lx2/episode/{eid}.json` | 直接返回结构化数据 |
| 音频文件 | `https://media.xyzcdn.net/{pid}/{hash}.m4a` | yt-dlp 下载 |

### 3.2 付费检测

列表页 description 中出现以下关键词之一，标记为付费集，跳过下载：
- `售价`、`购买`、`付费`、`单集.*元`、`优惠价`、`小鹅通`

付费集在列表中显示为**不可勾选**，右侧标签：`[需购买 ¥XX]`

## 4. 反爬策略

### 4.1 请求层

| 策略 | 实现 |
|------|------|
| 请求间隔 | **≥5 秒**（每次请求后等 5 秒再发下一个） |
| Cookie | UI 上提供输入框，用户从浏览器复制已登录 Cookie，定期更新 |
| 重试退让 | 失败后等 **10 秒** 再重试，最多 **3 次** |
| 频率上限 | 单次操作（获取15集）不超过 20 次 HTTP 请求 |

### 4.2 UA / Referer

- 每次请求随机从 3~5 个常见 UA 中选择
- 带上 `Referer: https://www.xiaoyuzhoufm.com/` 模拟浏览器跳转

### 4.3 降级方案

3 次失败后，切换 **Playwright**（无头浏览器）模拟真人访问。

## 5. 数据库（SQLite）

路径：`f:/voicetofile/voicetofile.db`

### podcasts 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增 ID |
| pid | TEXT UNIQUE | 小宇宙播客 ID |
| name | TEXT | 播客名称（抓取后缓存） |
| added_at | TIMESTAMP | 订阅时间 |

### episodes 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增 ID |
| podcast_id | INTEGER FK | 关联 podcasts.id |
| eid | TEXT | Episode ID |
| name | TEXT | 集名 |
| pub_date | TIMESTAMP | 发布时间 |
| duration | TEXT | ISO 8601 时长（PT169M） |
| is_paid | BOOLEAN | 是否付费 |
| status | TEXT | 见下方状态说明 |
| txt_path | TEXT | 转写结果 TXT 路径 |
| error_msg | TEXT | 失败时的错误信息 |
| created_at | TIMESTAMP | 首次添加时间 |
| updated_at | TIMESTAMP | 最后更新时间 |

### status 状态说明

| 状态 | 含义 |
|------|------|
| `pending` | 用户勾选了，等待下载 |
| `downloading` | 下载进行中 |
| `transcribing` | 转写进行中 |
| `done_deleted` | 转写完成，音频已删除 |
| `failed` | 失败，自动重试 2 次后仍失败则标记，等用户手动重试 |

## 6. 文件系统

### 6.1 目录结构

```
f:/voicetofile/
├── app.py                  # Flask 主程序
├── db.py                   # SQLite 数据库
├── scraper.py              # 小宇宙抓取 + 付费检测 + 反爬
├── downloader.py           # yt-dlp 音频下载
├── transcriber.py          # Faster-Whisper 转写
├── _utils.py               # 共享工具
├── ffmpeg/                 # 捆绑 ffmpeg（与 b-site 共享）
├── templates/
│   └── index.html          # 前端页面
├── static/
│   └── style.css           # 样式
├── voicetofile.db          # SQLite 数据库
└── 启动.bat                 # 双击启动
```

### 6.2 输出文件

用户配置的根目录（UI 可改，默认 `F:\outfile`）：

```
{根目录}/
└── {podcast_name}/
    ├── {episode_name}_文字稿.txt    # 转写结果
    └── {episode_name}_文字稿.txt      # 重新转写则覆盖
```

### 6.3 临时文件（清理对象）

- `output_dir/_audio_progress_{pid}.txt` — 转写进度
- `output_dir/_audio_result_{pid}.json` — 转写结果
- `output_dir/audio.wav` — 临时音频

### 7.4 刷新按钮功能

点击后：
1. 通过任务栏找到并关闭当前 Flask 进程（按端口或名称）
2. 清除 `temp/` 下的所有进度文件（`_audio_progress_*.txt`、`_audio_result_*.json`）
3. 重新启动 Flask
4. 自动打开浏览器 `http://127.0.0.1:18990`

## 8. 进度显示约定

各阶段 STATUS 消息格式（与 b-site 保持一致）：

```

▶ [45%] 正在提取音频... (12秒)
▶ [100%] 音频提取完成 (3.2秒)
```

队列列表实时三段显示：
- **下载阶段**：`{集名}  下载中  {进度}%  已用 {时间}`
- **转写阶段**：`{集名}  转写中  {进度}%  已用 {时间}`
- **等待中**：`{集名}  等待中`

## 9. 技术选型

| 组件 | 选型 | 说明 |
|------|------|------|
| Web 框架 | Flask | 轻量，与 b-site 一致 |
| 前端 | 原生 HTML/JS | 简单够用 |
| 数据库 | SQLite | 单文件，无依赖 |
| ASR | Faster-Whisper large-v3 | CUDA 加速，与 b-site 一致 |
| 下载器 | yt-dlp | 成熟方案 |
| 浏览器降级 | Playwright | 反爬最终降级方案 |
| ffmpeg | 捆绑版 | 与 b-site 共享，不依赖系统安装 |

## 10. 实现顺序

```
Step 1.  db.py          — 数据库结构建表
Step 2.  scraper.py     — 抓取 + 付费检测 + 反爬（Cookie/间隔/降级）
Step 3.  downloader.py  — yt-dlp 下载
Step 4.  transcriber.py — Faster-Whisper 转写
Step 5.  app.py         — Flask 主程序（路由 / SSE 推送 / 队列管理）
Step 6.  templates/     — 前端页面
Step 7.  启动.bat       — 双击启动脚本
Step 8.  _utils.py      — 共享工具（路径校验、文件名清理）
```

## 11. 已知边界情况

| 情况 | 处理方式 |
|------|---------|
| 付费集 | 列表展示，不可勾选，不下载 |
| 部分播客只有 1~2 集 | 正常处理，数量无下限 |
| 下载中途断网 | 标记 failed，自动重试 2 次 |
| 转写 OOM（GPU 显存不足） | 自动降级到 int8，仍失败则标记 failed |
| 标题过长（Windows MAX_PATH） | 自动截断标题后重试 |
| 音频已是 m4a（播客原生） | 跳过 ffmpeg 提取步骤，直接用原文件 |
| 重复添加同一集 | 提示"已在队列/已完成"，不重复添加 |
